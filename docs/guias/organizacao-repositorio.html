Passo 1: Criar a Estrutura de Pastas

No Terminal do VS Code

Abra o terminal integrado do VS Code (Ctrl + `) e execute os seguintes comandos:

# Criar pasta principal
mkdir keepla-repositorio
cd keepla-repositorio

# Criar estrutura de pastas
mkdir -p brand/assets/{logos,icons,images}
mkdir -p brand/guidelines
mkdir -p docs
mkdir -p src/{frontend,backend,mobile}
mkdir -p marketing/{social-media,email-templates,campaigns,analytics}
mkdir -p legal
mkdir -p scripts/{deployment,utilities}
mkdir -p tests/{unit,integration,user-acceptance}

# Inicializar repositório Git
git init

# Criar arquivo .gitignore
echo "node_modules/" >> .gitignore
echo ".env" >> .gitignore
echo "dist/" >> .gitignore
echo "*.log" >> .gitignore

Passo 6: Configuração de Versionamento

Configurar Git para o Projeto

Execute estes comandos no terminal do VS Code:

# Configurar usuário
git config user.name "Seu Nome"
git config user.email "seu.email@exemplo.com"

# Adicionar todos os arquivos ao staging
git add .

# Fazer primeiro commit
git commit -m "Initial commit: Setup project structure"

# Adicionar repositório remoto (se aplicável)
git remote add origin https://github.com/seu-usuario/keepla-repositorio.git

# Fazer push para o repositório remoto
git push -u origin main
Dica: Use a extensão "GitLens" no VS Code para ter melhor visualização e controle do seu versionamento.
